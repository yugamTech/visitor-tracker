## Hard rules (no exceptions, no "just this once")

1. **TypeScript strict.** Never use `any`. Use `unknown` and narrow. `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes` are on.
2. **No inline SQL.** All queries live in `src/db/queries/<entity>.ts`. The only exception is migrations.
3. **No DB access from client components.** Server actions and RSC only.
4. **`tenantId` is always explicit.** Every query function takes `{ tenantId }`. No global tenant context, no thread-locals, no "implicit current tenant".
5. **Zod is the source of truth.** Define schema in `src/lib/schemas/<entity>.ts`. Infer types via `z.infer<typeof X>`. Never hand-write a type that mirrors a schema.
6. **Server actions return `Result<T, E>`** (see ADR-0005). Never `throw` to the client. Wrap all DB calls in try/catch inside the action; convert to `{ ok: false, error }`.
7. **Env access only via `src/lib/env.ts`.** Never read `process.env.X` elsewhere. The CI step `pnpm lint` should flag this.
8. **`"use client"` at the lowest possible leaf.** Pages and layouts default to RSC. Drop to client only when you need state, refs, or browser APIs.
9. **One default export per component file.** Named exports are for utilities and hooks. Component filenames in kebab-case (`visit-request-card.tsx`); the default export is PascalCase (`VisitRequestCard`).
10. **No new dependency without an explicit prompt asking for it.** If you think one is needed, stop and propose it in your response.
11. **No new top-level folder.** The structure below is the structure.
12. **Migrations are append-only.** Never edit a migration that's been merged.
13. **Secrets never in the repo.** `.env.local` is gitignored. `.env.example` is the spec.
14. **Public routes do not access the session.** They still resolve tenant from the subdomain.
15. **Commit format:** Conventional Commits — `type(scope): subject`, lower-case subject, no trailing period. Allowed types: `feat | fix | refactor | chore | docs | test | ci | perf`.

## Coding patterns (these are the patterns; do not invent new ones)

### Query function

```ts
// src/db/queries/visit-requests.ts
import { and, eq, inArray } from 'drizzle-orm'
import { db } from '@/db/client'
import { visitRequests } from '@/db/schema'

export async function listPendingRequests(args: {
  tenantId: number
  userId: number
  groupIds: number[]
}) {
  return db.select().from(visitRequests).where(
    and(
      eq(visitRequests.tenantId, args.tenantId),
      eq(visitRequests.status, 'pending'),
      // routed directly to user OR to a group the user belongs to
      // (compose with `or(...)` — left as exercise for the implementer)
    ),
  )
}
```

### Server action

```ts
// src/actions/visit-requests/approve.ts
'use server'

import { z } from 'zod'
import { requireRole } from '@/lib/auth-helpers'
import { ok, err, type Result } from '@/lib/result'
import { AppError } from '@/lib/errors'
import * as q from '@/db/queries/visit-requests'

const InputSchema = z.object({
  requestId: z.number().int().positive(),
  note: z.string().max(500).optional(),
})

export async function approveVisitRequest(
  raw: unknown,
): Promise<Result<{ requestId: number }, AppError>> {
  const parsed = InputSchema.safeParse(raw)
  if (!parsed.success) return err(AppError.validation(parsed.error.flatten()))

  const user = await requireRole('admin', 'super_admin')

  try {
    const updated = await q.approveRequest(parsed.data, {
      tenantId: user.tenantId,
      actorUserId: user.id,
    })
    return ok({ requestId: updated.id })
  } catch (e) {
    return err(AppError.system(e))
  }
}
```

### Component (RSC default)

```tsx
// src/app/(app)/admin/page.tsx — RSC, no "use client"
import { requireRole } from '@/lib/auth-helpers'
import * as q from '@/db/queries/visit-requests'
import { PendingRequestsList } from '@/components/admin/pending-requests-list'

export default async function AdminHome() {
  const user = await requireRole('admin', 'super_admin')
  const pending = await q.listPendingRequests({
    tenantId: user.tenantId,
    userId: user.id,
    groupIds: user.groupIds,
  })
  return <PendingRequestsList items={pending} />
}
```

The client component (`PendingRequestsList`) is the leaf — it has `"use client"`.
